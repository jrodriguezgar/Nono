# Summarization Reference

## Techniques by Content Type

| Content | Technique | Key Considerations |
|---------|-----------|-------------------|
| **News articles** | Inverted pyramid — lead with the conclusion | Who, what, when, where, why |
| **Technical docs** | Structured hierarchy — mirror section headings | Preserve technical terms |
| **Meeting notes** | Action-oriented — decisions, owners, deadlines | Skip small talk, keep commitments |
| **Research papers** | IMRAD — objective, method, findings, implications | Keep numbers and stats |
| **Legal documents** | Clause-based — obligations, rights, conditions | Never paraphrase legal terms |
| **Emails/messages** | Intent-based — request, response, information | Identify action items |

## Length Guidelines

| Source Length | Summary Target | Key Points |
|--------------|---------------|------------|
| < 500 words | 1–2 sentences | 2–3 |
| 500–2000 words | 1 paragraph | 3–4 |
| 2000–5000 words | 2–3 paragraphs | 4–5 |
| > 5000 words | Multi-section summary | 5 (maximum) |

## Quality Checklist

- [ ] Captures the main argument or conclusion
- [ ] Includes all critical facts, figures, and names
- [ ] Does not add information absent from the source
- [ ] Maintains neutral tone (no interpretation)
- [ ] Uses clear, concise language
