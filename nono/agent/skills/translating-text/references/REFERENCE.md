## Translation Quality Guidelines

| Principle | Description |
|-----------|-------------|
| **Accuracy** | Meaning must be preserved — no additions or omissions |
| **Fluency** | Output must read naturally in the target language |
| **Terminology** | Technical terms should stay consistent throughout |
| **Formatting** | Preserve bullet points, headers, emphasis, code blocks |
| **Tone** | Match the formality and style of the source |

## Common Pitfalls

- **False friends**: Words that look similar across languages but have different meanings.
- **Literal translation**: Avoid word-for-word translation of idioms and expressions.
- **Gender/formality**: Some languages require formal/informal distinctions (e.g., tú/usted, tu/vous).
- **Right-to-left**: Arabic and Hebrew require RTL text direction awareness.
- **Pluralization rules**: Different languages have different plural forms (e.g., Russian has 3).
